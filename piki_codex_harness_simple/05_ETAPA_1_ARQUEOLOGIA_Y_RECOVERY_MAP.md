# Etapa 1 — Arqueología y recovery map

## Objetivo

Analizar los dos TXT y convertir el legado en un mapa verificable.

## Trabajo

- detectar marcadores de archivos;
- indexar slim y grande;
- localizar runtime real y tests;
- distinguir código de documentación;
- crear recovery map;
- localizar Jinja, fusión, active topic, tools, fotos y grounding;
- detectar secretos sin copiarlos;
- validar `INTERNAL_API.md`.

## Entregables

```text
docs/LEGACY_RECOVERY_MAP.md
docs/GOLDEN_BEHAVIORS.md
docs/ARCHITECTURE_DECISIONS.md
```

## Gate

- no runtime implementado;
- recovery map con archivo, evidencia, clasificación y destino;
- lista explícita de descartes;
- API real comprendida;
- Meta oficial registrada como reemplazo de Baileys;
- riesgos y contradicciones documentados.
